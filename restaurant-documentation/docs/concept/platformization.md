# Платформизация курсового проекта

## 1. Функциональность, которую можно реализовать с помощью существующих решений на рынке

|Функциональность|Почему заменять|Решение на рынке|Где применять|
|-|-|-|-|
|Картография и геолокация|Разработка собственного картографического сервиса — гигантская задача требующая огромных вычислительных мощностей. Нет необходимости изобретать велосипед, когда на рынке существуют сервисы (Яндекс.Карты, 2ГИС), которые уже успешно реализуют весь необходимый функционал геолокации.|Яндекс.Карты|На странице поиска ресторанов для отображения найденных/отфильтрованных ресторанов, так же подбор ресторанов исходя из геолокации пользователя.<br/>На странице детального описания ресторана для отображения ресторана на карте.|
|Отправка уведомлений (Email, SMS)|Самостоятельная настройка отправки email/SMS требует решения проблем доставляемости, очередей, повторных попыток. Готовые сервисы предоставляют удобные API и аналитику.|Unisender|Для нотификаций об успешном бронировании, отмене брони, напоминания о предстоящей брони|
|Авторизация через соцсети|Пользователям может быть удобнее входить через существующий аккаунт. Предоставить выбор пользователю авторизоваться в приложении через телефон и пароль или через удобный для них аккаунт, чтобы можно больше пользователей авторизовывались и начинали пользоваться системой|Yandex OAuth, VK OAuth|При регистрации в приложении|
|Аналитика и веб-метрики|Анализ поведения пользователей (откуда пришли, что искали, где нажимали) — отдельная сложная задача, требующая специализированных инструментов. Как и в случае с геолокацией, на рынке уже разработаны сервисы для решения задач аналитики поведения пользователей|Яндекс.Метрика|На сайт добавляется фрагмент JavaScript-кода («счётчик»). Сервис сам собирает и отправляет данные, предоставляя готовые отчёты по действиям пользователей.|

## 2. Стратегия платформизации приложения

Приложение является посредником между ресторанами и посетителями, которые хотят забронировать столик. Модель бизнеса **B2B2C** (Business-to-Business-to-Consumer).

|Сегменты|Ценность|Монетизация|
|-|-|-|
|Рестораны, бары, кафе, ресторана|Ресторан получает доступ к готовой системе онлайн-бронирования, которая привлекает новых гостей, снижает число неявок за счёт автоматических напоминаний и освобождает персонал от приема заявок по телефону.|Основной источник монетизации|
|Платформы доставки, такие как Яндекс.Еда|Сервис доставки, интегрируя нашу платформу бронирования, получает конкурентное преимущество перед аналогами: пользователь может не только заказать еду, но и сразу забронировать столик в том же ресторане. Это повышает лояльность и удержание клиентов, а также даёт дополнительный доход от комиссии с бронирований.|Дополнительный источник монетизации|
|Посетители|Поиск ресторанов по кухне, цене, рейтингу, местоположению. Быстрое и удобное он-лайн бронирование/отмена брони столиков. Возможность выбрать столик и сделать прибывание в ресторане наиболее комфортным.|Для них сервис бесплатный|

Стратегия монетизации:
|Способ|Описание|
|-|-|
|Подписка (SaaS)|Рестораны платят ежемесячную подписку за использование платформы|
|Комиссия с бронирования|Ресторан платит процент или фиксированную сумму за каждую успешную бронь, совершённую через платформу|
|Плата за премиум-функции|Выделение ресторана в поиске, продвижение карточки ресторана, приоритетные показы|
|Плата за API|Ежемесячный платёж за доступ к API бронирования столиков для партнеров таких как Яндекс.Еда|
