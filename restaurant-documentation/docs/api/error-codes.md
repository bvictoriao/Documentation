## HTTP-статусы

| Код | Статус | Когда возникает |
| --- | ------ | --------------- |
| 200 | OK | Успешное получение, обновление или отмена брони |
| 201 | Created | Успешное создание брони |
| 400 | Bad Request | Ошибка валидации входных данных |
| 401 | Unauthorized | Пользователь не авторизован |
| 403 | Forbidden | Недостаточно прав (например, доступ к admin-методам) |
| 404 | Not Found | Ресурс по указанному `id` не найден |

## Коды ошибок в теле ответа

При ошибке API может вернуть JSON с полями `code` и `message`:

```json title="Пример ответа 400"
{
  "code": "VALIDATION_ERROR",
  "message": "Поле guests должно быть больше 0"
}
```

| Код ошибки | HTTP | Описание |
| ---------- | ---- | -------- |
| `VALIDATION_ERROR` | 400 | Не переданы обязательные поля или нарушены ограничения |
| `UNAUTHORIZED` | 401 | Пользователь не прошел аутентификацию |
| `FORBIDDEN` | 403 | Роль пользователя не позволяет выполнить операцию |
| `RESTAURANT_NOT_FOUND` | 404 | Ресторан с указанным `id` не найден |
| `BOOKING_NOT_FOUND` | 404 | Бронь с указанным `id` не найдена |
| `TABLE_NOT_AVAILABLE` | 409 | Столик недоступен на выбранные дату и время |
