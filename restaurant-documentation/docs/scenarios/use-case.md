# Use Case диаграмма

Диаграмма показывает роли пользователей и ключевые сценарии в сервисе бронирования столиков.

```plantuml
@startuml
left to right direction

actor "Незарегистрированный\nпользователь" as Guest
actor "Авторизованный\nпользователь" as User
actor "Администратор\nресторана" as RestAdmin
actor "Администратор\nплатформы" as PlatformAdmin

Guest <|-- User

rectangle "Сервис бронирования столиков" {
  usecase "Зарегистрироваться\nв системе" as UC1
  usecase "Войти в систему" as UC2
  usecase "Найти ресторан\nпо фильтрам" as UC3
  usecase "Просмотреть информацию о \nресторане" as UC4
  usecase "Выбрать столик\nна схеме зала" as UC5
  usecase "Выбрать дату\nи время" as UC6
  usecase "Забронировать\nстолик" as UC7
  usecase "Проверить доступность\nстолика" as UC8
  usecase "Подать заявку\nна регистрацию ресторана" as UC9
  usecase "Зарегистрировать\nресторан" as UC10
  usecase "Проверить данные\n ресторана" as UC17
  usecase "Активировать\n профиль ресторана" as UC18
  usecase "Отказать\n в регистрации" as UC19

  usecase "Просмотреть данные\n о столиках" as UC11
  usecase "Просмотреть список\n броней" as UC12
  usecase "Отменить\n бронь" as UC13
  usecase "Управлять профилем\n ресторана" as UC14
  usecase "Управлять схемой\n зала и столиков" as UC15
  usecase "Просмотреть бронирования" as UC16

  UC7 ..> UC5 : <<include>>
  UC5 ..> UC6 : <<include>>
  UC6 ..> UC8 : <<include>>
  UC4 <.. UC11 : <<extend>>
  UC10 ..> UC17 : <<include>>
  UC17 <.. UC18 : <<extend>>
  UC17 <.. UC19 : <<extend>>
}

Guest --> UC1
Guest --> UC2

User --> UC3
User --> UC4
User --> UC12
User --> UC13
User --> UC7

RestAdmin --> UC9
RestAdmin --> UC14
RestAdmin --> UC15
RestAdmin --> UC16

PlatformAdmin --> UC10

@enduml
```

## Краткое описание

- Гость может зарегистрироваться и войти в систему.
- Авторизованный пользователь ищет ресторан, просматривает карточку, выбирает столик/время и создает бронь.
- Администратор ресторана подает заявку, управляет профилем и схемой зала, отслеживает бронирования.
- Администратор платформы проверяет данные ресторана и принимает решение о регистрации.
