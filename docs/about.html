Документация проекта «Онлайн-бронирование столиков»

О проекте​

Сервис помогает пользователям находить рестораны по предпочтениям (кухня, цена, расположение, заполненность) и бронировать столики онлайн. Рестораны получают готовую систему бронирования и дополнительный поток гостей.

  • Бизнес-требования (проблема, аудитория, цели, риски)
  • Границы MVP и план развития

Навигация по артефактам системного анализа​

Требования и стейкхолдеры​

АртефактОписание
Сбор требованийКлассификация стейкхолдеров, матрица RACI для функционала бронирования, вопросы для интервью
Функциональные требованияОписание функциональных требований в формате Use Case
Нефункциональные требованияОписание нефункциональных требований и приоритезация на основе фрейм ворка MoSCoW

Модели и спецификации​

АртефактОписание
BPMN диаграммаОсновной и альтернативные потоки онлайн-бронирования (UC-004). Включена синхронная проверка API ресторана, резервирование, сохранение в БД, асинхронное уведомление.
Use Case диаграммаДиаграмма показывает роли пользователей и ключевые сценарии в сервисе бронирования столиков.
Sequence диаграммаДиаграмма отражает процесс бронирования столиков.
Макет интерфейсаСсылка на интерактивный макет в Unidraw, скриншоты экранов и описание источников данных.
APICпецификация OpenAPI.
База данныхСущности (User, Restaurant, Table, Booking, RestaurantRegistration), атрибуты, объёмы данных, паттерны доступа, выбор реляционной СУБД + Object Storage для фото.
ERDERD диаграмма, описание сущностей.
Асинхронное взаимодействиеОбоснование выбора RabbitMQ для отправки нотификаций (email/SMS) после успешного бронирования.
ПлатформизацияИспользование готовых решений (Яндекс.Карты, Unisender, OAuth, Яндекс.Метрика). Стратегия B2B2C и монетизация (подписка, комиссия, премиум, плата за API).
